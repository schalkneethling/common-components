Project Goal Summary

Fiori should become a practical, accessible web-component library that pushes the web platform while documenting production readiness, browser support, Baseline feature availability, polyfills, dependencies, and integration paths. It should also act as a real-world proving ground for Miyagi, Project Calavera, CSS validation, axe reporting, React usage, and agent tooling.
